================================================================================
                          快速开始
================================================================================

1. 创建数据源：
   POST /api/data-sources
   {
     "name": "生产MySQL库",
     "db_type": "mysql",
     "host": "192.168.1.1",
     "port": 3306,
     "username": "admin",
     "password": "secret123",
     "database_name": "prod_db",
     "description": "线上数据库"
   }

   响应：data_source_id (UUID7)

2. 创建会话时引用数据源：
   POST /api/sessions
   {
     "user_id": "user-uuid7",
     "data_source_id": "data-source-uuid7"
   }

3. 列表数据源：
   GET /api/data-sources?skip=0&limit=10

4. 更新数据源：
   PUT /api/data-sources/{id}
   {
     "name": "新名称",
     "is_active": "false"
   }

5. 删除数据源：
   DELETE /api/data-sources/{id}

================================================================================
                      数据库表结构
================================================================================

data_sources
├── id (VARCHAR(36) PK)          - UUID7
├── name (VARCHAR(255))          - 数据源名称
├── db_type (VARCHAR(50))        - 数据库类型
├── host (VARCHAR(255))          - 主机名/IP
├── port (INTEGER)               - 端口号
├── username (VARCHAR(255))      - 用户名
├── password (VARCHAR(255))      - 密码
├── database_name (VARCHAR(255)) - 数据库名称
├── description (TEXT)           - 描述
├── is_active (VARCHAR(10))      - 激活状态
├── created_at (DATETIME)        - 创建时间
└── updated_at (DATETIME)        - 更新时间

chat_sessions (修改)
├── id (VARCHAR(36) PK)          - UUID7
├── user_id (VARCHAR(36) FK)     - 用户ID
├── data_source_id (VARCHAR(36) FK) - 数据源ID
├── status (VARCHAR(20))         - 会话状态
├── log_filename (VARCHAR(255))  - 日志文件名
├── log_filepath (VARCHAR(512))  - 日志路径
├── created_at (DATETIME)
└── updated_at (DATETIME)

数据源管理：

聊天功能（端点不变）：
